Content Format Taxonomy¶

The content format taxonomy is used to manage the different types of legal content. The content format is set automatically by the system.

Warning

Do not add new terms to this taxonomy without telling Gwen. They will not be automatically associated to legal content without a code update.

The content format is added to legal content in the “primary content type” field based on the content blocks used in the piece of content. The rules for determining primary content type are as follows and once a match is found, the system stops processing (so an article that meets rule 1, no other rules can overwrite it).

The content type is set to:

  1. IICLE if it contains an IICLE block
  2. Easy form if it contains an automated document block
  3. Teach me if it contains a teach me block
  4. Guide if it contains a legal bundle block
  5. How-to if it contains a process step
  6. Form - download if it contains a static form block AND that form has a file attached
  7. Form - link if it contains a static form block AND that form has a URL field in it
  8. Link - if it contains a link content block
  9. File - if it contains a file content block
  10. Video - if it contains a video block
  11. Text article - if no other rule has fired, it is treated as a text article.

Note

These rules stop with the first match. So an article that contains an IICLE block, a video, and a file will be classified as an IICLE; an article with a video and a text block will be treated as a video. An article with a process step and a text block would be treated as a How-to but an article with a process step and a legal bundle would be treated as a Guide.

These content formats are then used as labels on the legal information and search results pages.
