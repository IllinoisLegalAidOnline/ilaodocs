ILAO person custom data¶

ILAO captures the email address of users as the identity piece. Email comes from:

  • Mail field for registered users
  • From email fields for anonymous users:
    • Contact us form
    • Online intake contact form
    • Newsletter subscription form
    • Blog subscription form

We can have up to 50 additional user defined variables for “person.” We have defined the following as additional user-defined variables:

UDF 1 to 3 are already in use by the default configuration.

Variable name Definition Type and Location
person_udf4 Company Organization/Company/Law school (registered) String Profile
person_udf5 Editor Content editor role Indicates whether the user is a SME or has edited a piece of legal content Boolean Profile
person_udf6 Organization Role Organization role Indicates whether user is org admin, OTIS admin, or both. Comma-list Profile
person_udf7 Zip code Zip code String Profile
person_udf8 Gender Gender String Profile
person_udf9 Population Segments Populations Comma-list Profile
person_udf10 Notification Preferences Notification preferences For logged in users whether they want email/sms Comma-list Profile
person_udf11 Year born Year born. Available for online intake and registered users Integer Profile
person_udf12 Primary role Primary registered user role Available for registered users only. We store the best role for a user: Intern, if has intern role Staff if has staff role and not intern All others: advocate, pro bono, law student, navigator, public String Profile
person_udf13 User tester User testing status (registered users only Boolean Profile
